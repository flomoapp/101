# 试试用 flomo + excel，搭建自己的工作流｜阿粒

上一篇讲到，光滑则意味着迅速、熟悉；熟悉意味着忽略细节，而忽略细节则代表自己思维成为定势。意在强调，思考时不要过于顺滑，而是去感受碰撞和摩擦。

slow is faster.

但，任何时候都要增加摩擦力吗？

想象一下，如果你正打算前往一处地方，结果隔几秒就会被别人撞一下，前进受阻，心情恐怕也会暴跳起来。

# 为什么要减小摩擦力？

  • 让行动更顺畅
  • 减少自我损耗

# 什么时候减小摩擦力

  • 信息输出

信息论中提到一个观点，“万物皆比特”。比特是信息的单位。这句话我理解为，“万物皆信息”。所以这里的「信息输出」，不仅仅是指写一篇文章，也可以指 设计一次旅行、开发一个产品、组织一次活动等等。

# 怎么减少行动的摩擦力

这个话题比较宏大，与之相似的问题很多。比如，知乎上的热门问题：「如何从空有上进心的人，变成行动上的巨人」、「怎么样才能做到让自己立刻行动」、「有什么逼自己立刻行动的方法」...

民间方法很多，尽管选取你当下就能接受的，然后马上试试看。弱水三千，取一瓢即可。

这里和大家分享我的2个原则，不多，但足够我自己使用。这两个原则，取自我在flomo里面的2个标签。

# 2个原则：颗粒度+工程化

# 颗粒度

  • 信息拆解颗粒度越小，组装的灵活度更高，组装可能性更多；
  • 任务拆解颗粒度越细，推动的摩擦力更小，积累的惯性势能越大；
  • 做事颗粒度越细，越能做成一件事情。

之所以关注到「颗粒度」，是因为 「接触到flomo原子化的设计理念」、「读到了脱不花的四个面试题」、「之前看过的L先生的一篇文章」...，这些原本零散的信息点，最终凝聚在了这个标签下面。

感兴趣的可以复制链接到浏览器打开：

从减少摩擦力这个角度来看：「任务拆解颗粒度越细，推动的摩擦力更小，积累的惯性势能越大」。

# 工程化

「工程化」这个词，摘录自fonter的一篇文章。

工程化你的习惯fonter，公众号：耳聽工程化你的习惯 (opens new window)

用直白一些的话来描述，可以理解为：「给你要做的事情搭一个管道」，让事情自己顺着管道滑下去。背后的原因是：

  • 假设未来的自己是懒惰的，永远在想，下一次做这件事情怎么样更轻松
  • 我有一个很懒的习惯，凡是一件事，只要能让我重复的事情做两遍以上，我就会偷懒，我就会想办法去偷懒。你用工具也好，脚本也好，自动化的工具也好，我一定会想办法去偷懒。
  • 因懒而生的勤奋，经常创造思想和发明；因勤奋而产生的懒，令人放弃真正的思考。

# 1个实践：flomo+excel

「颗粒度」和「工程化」，是减少摩擦力的两个原则。具体在方法层面，和大家分享一个最近的实践：

用 flomo+excel，搭建自己的工作流。

# flomo #workday：

承担速记的功能，记录这一天工作发生的大小事情，不管是反馈的需求、还是出现的bug、或者是会议的共识等等。每天一张卡片，标注 day XX，统一在事情发生之后，快速记录在这里。

工作间隙休息时，或者下班前，回看一遍，把信息点再分类流转到不同的excel里面。比如 新反馈的需求，进入到「excel_用户story」中。

另外下班前列出一些待确认事项，放到第2天的 flomo #workday里，作为早上站立会的沟通点。

# 为什么有flomo #workday？

每天零零碎碎的信息很多，不能够让自己老是被打断，靠脑子记忆肯定会遗漏。所以一旦出现新信息，先丢进今天的workday再说，反正下班前会再复一遍（写周报的时候也会）。这样既不会遗漏，也避免对自己的过分打扰。

# excel #workday

一周5天，作为一个单元。左侧是产品/项目的本周重点（从excel #week 的下周计划中复制过来），右侧是落到每天的计划安排。

需要注意的是，无论是预先安排当天工作重点、或者是记录当天核心进度、或者是记录当天发生的重大问题，都必须注意简明扼要，只记录「线索」即可，不要太多细节。因为excel空间不大，写太多文字会丧失可读性。

# 为什么有excel #workday？

之前在计划下周工作的时候，颗粒度很粗。就是你只知道要做一个事情，但是具体分哪些步骤，是没有写出来的。这样糟糕的地方就是，一是没有办法把控进度，容易黄；二是每天还得想从大的本周工作里面去摘取明天要做什么，又累又焦虑。重要的事情都按照工作日放好位置了，一眼看过去都在，可以给到确定感和安全感。

# excel #weekly：

汇总本周的重要事项及进度，也就是周报。左侧是时间维度，顶部是项目/产品维度。我喜欢按照M3W2这样的命名方式，表示3月份的第2个礼拜。D表示done，本周完成；P表示plan，下周计划。当前进行中的版本，同步其本周的进度是否正常、本周重要信息（比如重大变更/问题）。特别重要的用红色标注。

# 为什么有excel #weekly？

因为需要写周报、月报、季度报。写周报时的依据，就是本周的 flomo #workday 、excel #M5 。最新的周报，放在最上面。这样从上到下浏览的时候，就是本年/本季度/本月的历史周报，不用去翻邮件。excel写完，也可以直接复制到邮件里，不用再重新编辑。

总结来说，增加思考的摩擦力，减少行动的摩擦力。写到这里，想起了很古早的一句话：不要用战术上的勤奋，掩盖战略上的懒惰。也可理解为：战略上增加摩擦力，战术上减少摩擦力。

以上，希望对你有启发。
