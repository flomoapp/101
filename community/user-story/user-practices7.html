# 我的 flomo 心得，及Notion、语雀、flomo 如何搭配使用？｜JefferyHo

享我的几个数据：使用flomo 27 天，累计 57 个memo，连续达成率100%。

下面会从WHY-HOW-WHAT来讲一下我在flomo这些天的使用体验。

# #1 WHY flomo？

首先说下背景，我是一名体验设计师，日常记录想法💡对我来说属于思维沉淀的一个过程。

我最开始应该是在产品沉思录上看到这款产品，其实一直苦于寻找一款足够“轻”的记录工具，在“轻”这一点上，我认为flomo做到了以下两点：

1.“记录”这个行为本身足够轻

flomo可以与我们的微信进行联动，通过订阅flomo的服务号，我们就可以像给文件传输助手发消息一样把内容发布到我们的flomo信息流上。

这个功能，重新调整了我们的记录路径，从以往的：打开某个app或者网页——找到发布按钮——点击输入框——开始编辑。类似这样的一个流程，调整到 ：微信内找到flomo服务号——点击进入——点击输入框——开始编辑 。

如果你刚好把flomo服务号置顶在你的聊天列表上，那么路径上还可以省去”通过搜索寻找flomo“这一步；这样的一个操作路径，与我们日常的微信聊天方式非常类似，我觉得这是flomo这个功能做得很取巧的一步，正因为很接近我们的日常习惯，所以才显示足够”轻“，非常感谢flomo团队的努力。

2.“查看”这个行为足够轻

在flomo上，我们所有的消息都直接以信息流的形式呈现在首页，你说关于消息主体这块儿，还有没有二级页面？没有，就是这么简单直接。

除此之外，flomo左上角的成就网格，每天只要写上一个memo，就可以点亮一个格子，并且当天记录越多，格子颜色也会越深，我可以直观看到我的记录从无——有——多的变化过程，并且这个位置在用户视觉动线的起点，用户每次进来第一时间都会看到这个成就，可以刺激用户多做记录。

以上的“轻”和“成就”，就是我开始使用flomo的理由。

# #2 HOW flomo？

我日常的使用场景大多发生在移动环境中，因此我用得最多的是微信输入和app(严格来说是PWA)上，对我来说，记录一般会发生在早晚通勤、午间和周末午间，记录内容必须遵循以下特点：

1.原创，非原创的想法对我来说记录下来没有意义，这些想法通常来源于我对环境、行业、职业、个人的思考；

2.感兴趣，我认为要开始一件事情首先是需要对它“感兴趣”，当我开始对某几个领域感兴趣之后，我才会以这些领域为圆心，画圆，把这个圆，不断扩大；

3.无负担，我每天只会记录1-2条，这是现阶段最符合我的生活节奏的一个数量，我认为“记录”这个事情本身必须无负担，当我觉得我需要记录我的想法，我会打开flomo，这是“习惯”；但如果我为了保持记录而“提醒”了自己去开始创造想法并且记录，这就成了“任务”，也就成为了一个有终点的计划。我认为这对“想法”本身，不是好的“养成”方式。

# 3 WHAT flomo？

我目前用于记录的工具有三个：

Notion：用于编辑我的个人博客(jefferyho.com)，并且对外发布，这里会有我经过沉淀之后的思考，属于“重”型记录工具；

语雀：用于搜集我日常感兴趣的文章，以及发布我个人结构化后的思考，使用它是因为它在文档编辑的格式支持上很友好，也支持markdown，并且注重“知识库”的概念，我很尊重语雀团队对这个形态所做的一切；

flomo：承担了我所有的零碎想法，这些想法首先需要被记录下来，然后才能形成结构性的输出，它就是我写作的“素材库”。
